# Contributing Guide

## Workflow

1. Create an issue or pick an assigned issue.
2. Create a branch from `main`.
3. Keep your change focused.
4. Use meaningful commit messages.
5. Open a pull request early.
6. Ask for review from a classmate.
7. Address comments on the same branch.
8. Merge only after approval.

## Commit examples

- `fix: correct homepage heading typo`
- `feat: add empty state for no tasks`
- `docs: improve local setup steps`

## Pull request checklist

- [ ] My branch is up to date with `main`
- [ ] I linked the related issue
- [ ] I tested my change locally
- [ ] I added screenshots if the UI changed
- [ ] I responded to review comments

## Review checklist

- Is the change correct?
- Is the scope small and focused?
- Are naming choices clear?
- Are there any edge cases missing?
- Is the PR description complete?
