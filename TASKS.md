# Suggested classroom issues

Create these as GitHub issues and assign them to students.

## Beginner tasks

1. Fix the `Welcom` typo in the homepage heading.
2. Improve the README run instructions.
3. Add a footer to the homepage with a support email.
4. Change the `Add task` button text to `Add item`.
5. Add an empty-state message when there are no tasks.
6. Add a character counter below the input field.
7. Add a `Clear completed` button.
8. Add a confirmation before deleting a task.
9. Improve mobile spacing around the action buttons.
10. Add comments explaining the `renderTasks` function.

## Intermediate tasks

11. Refactor repeated DOM selectors into constants.
12. Prevent duplicate tasks from being added.
13. Sort completed tasks to the bottom.
14. Save tasks to `localStorage`.
15. Add a simple filter: `All`, `Open`, `Completed`.
16. Update the PR template to include a rollout risk section.
17. Improve accessibility labels on form elements.
18. Add keyboard support so Enter submits the form.
19. Add a lightweight test checklist to the README.
20. Add issue labels and triage guidance to `CONTRIBUTING.md`.

## Review-only tasks

21. Review a classmate's PR and request changes with at least 2 specific comments.
22. Approve a classmate's updated PR after feedback is addressed.
23. Leave one suggestion comment using GitHub's suggested change feature.
24. Resolve one merge conflict and document what happened in the PR.

## Stretch tasks

25. Add GitHub Actions to run a basic HTML/CSS/JS check.
26. Convert the issue templates to GitHub issue forms.
27. Add a `SECURITY.md` file.
28. Create a release note draft for the merged PRs.
